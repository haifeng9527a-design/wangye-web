# 分时图：时间轴与 Y 轴修改方案（对齐主流行情软件）

## 一、现有数据与 API（不改动）

| 数据 | 来源 | 说明 |
|------|------|------|
| 分时 K 线 | Polygon Aggregates / 后端 `GET /api/candles?symbol=X&interval=1min\|5min\|15min\|1h` | 每根含 `t`(时间戳)、`o/h/l/c`、`v`(成交量) |
| 昨收 | Snapshot / getPreviousClose / 详情 realtime 接口 | `prevClose` |
| 当前价 | 实时接口 / WebSocket | `currentPrice` |
| 周期 | 前端已支持 | 1m / 5m / 15m / 1h / 1D |

以上已满足「时间轴按真实时间」「Y 轴价格 + 右侧涨跌幅」的展示需求，无需新增接口。

---

## 二、别人软件 vs 我们现状

### 2.1 时间轴（X 轴）

| 项目 | 别人 | 我们现状 | 目标 |
|------|------|----------|------|
| 格式 | 日期 + 整点/半点：`02/25`、`10:30`、`11:30`、`12:30`、`13:30`、`02/25 14:20`、`15:30`、`16:00` | 仅 5 个等分点，格式 `HH:mm` 或 `MM/dd` | 与别人一致：左侧首格日期，中间为整点/半点时间，悬浮点可带日期+时间 |
| 刻度选取 | 按「真实时间」取整（如 10:00、10:30、11:00…） | 按「根数等分」取第 0、1/4、2/4、3/4、最后一根 | 按真实时间取整，再映射到最近一根 K 线的索引 |
| 数量 | 约 6～8 个刻度 | 固定 5 个 | 5～8 个，随区间长度微调 |

### 2.2 Y 轴（左侧价格）

| 项目 | 别人 | 我们现状 | 目标 |
|------|------|----------|------|
| 刻度 | 价格等分，约 5～7 档 | 5 档，等分 minY～maxY | 保持 5～7 档等分 |
| 小数位 | 2 位（如 121.37） | 2 位 | 保持 |
| 当前价高亮 | 当前价所在刻度用绿框标出 | 无 | 当前价刻度用背景色/边框高亮（有 currentPrice 时） |

### 2.3 Y 轴（右侧涨跌幅）

| 项目 | 别人 | 我们现状 | 目标 |
|------|------|----------|------|
| 基准 | 昨收为 0% | 已用 prevClose 算百分比 | 保持 |
| 刻度 | 与左侧价格一一对应，显示百分比 | 已对应 | 保持 |
| 当前值高亮 | 当前涨跌幅所在刻度用绿框 | 无 | 与左侧一致，当前%刻度高亮（有 currentPrice + prevClose 时） |

## 三、具体修改方案（仅用现有 API 数据）

### 3.1 时间轴（X 轴）

- **输入**：`candles`（每根含 `time` 秒时间戳）、`periodLabel`（1m/5m/15m/1h/1D）。
- **逻辑**：
  1. 取时间范围：`t0 = candles.first.time`，`t1 = candles.last.time`（或当前时间）。
  2. 按周期决定步长与格式：
     - 1D：按「天」取整，显示 `MM/dd`。
     - 1h/15m/5m/1m：在 [t0, t1] 内，生成「整点或半点」时间点（如 09:30、10:00、10:30、11:00…），再映射到**最近一根 K 线的索引**，得到要显示的索引列表。
  3. 首格：若当天有多根 K 线，第一个标签用日期 `MM/dd`（或 `MM/dd HH:mm`），其余用 `HH:mm`；若跨日则可在日期变更处再打一次 `MM/dd`。
  4. 展示：时间轴下只在这些「映射到的索引」位置绘制刻度标签，避免等分导致的「时间不对齐」感。
- **数据**：仅用 `candles[].time`，不新增接口。

### 3.2 Y 轴（左价格 + 右涨跌幅）

- **输入**：`minYPlot`、`maxYPlot`（含上下留白）、`basePrice = prevClose`、`currentPrice`（可选）。
- **逻辑**：
  1. 左侧：在 [minYPlot, maxYPlot] 上等分 5～7 档，每档格式化为价格（2 位小数）。
  2. 右侧：同一档位用 `(v - basePrice) / basePrice * 100` 得到百分比，格式 `+x.xx%` / `-x.xx%`。
  3. **当前价高亮**：在左侧找到与 `currentPrice` 最接近的刻度线，该行整行（左价格 + 右百分比）用浅色背景或边框高亮（如浅绿/主题色）；若无 `currentPrice` 则不画。
- **数据**：仅用 `prevClose`、`currentPrice`（详情页已有），不新增接口。

### 3.3 可选：均价线

- **数据**：用现有 `candles` 的 `c`、`v` 计算当日均价（VWAP 或简单均线）。
  - 简单均价：`avg = sum(close) / n`。
  - VWAP：`vwap = sum(close * volume) / sum(volume)`（按 bar 累加）。
- 在主图用另一条线（如橙色虚线）绘制，与别人「均」线一致；无新 API。

### 3.4 可选：十字线（悬浮显示时间、价、涨跌幅）

- 交互：手指/鼠标悬浮或长按时，显示十字线 + 当前点 `时间`、`价格`、`涨跌幅`。
- 数据：当前点对应 `candles[index]` 的 `time`、`close`，以及 `prevClose` 算出的 `(close - prevClose) / prevClose * 100`，全部为已有字段。

## 四、实现优先级建议

| 优先级 | 项 | 说明 |
|--------|----|------|
| P0 | 时间轴按「真实时间」取整并映射到 K 线索引 | 时间轴观感与别人一致，仅改刻度选取与格式 |
| P0 | Y 轴左侧价格 + 右侧涨跌幅「当前价/当前%」高亮 | 用现有 currentPrice、prevClose 即可 |
| P1 | 时间轴首格显示日期、跨日处补日期 | 多日/跨日时更清晰 |
| P1 | 主图叠加均价线（简单均线或 VWAP） | 用 candles 计算，无新 API |
| P2 | 十字线悬浮显示时间/价/涨跌幅 | 纯交互与现有数据 |

---

## 五、涉及文件（建议）

- **时间轴 + Y 轴刻度与高亮**：`lib/features/market/chart/intraday_chart.dart`
  - 新增：按真实时间生成 X 轴刻度索引与文案（`_formatTime` 扩展、按整点/半点取点）。
  - 修改：左侧/右侧 Y 轴由「等分 5 档」改为「等分 5～7 档」，并增加「当前价/当前%」所在行的背景或边框高亮。
- **均价线（可选）**：同一文件内用 `candles` 计算一条均线/VWAP 序列，加一条 `LineChartBarData`（虚线）。
- **十字线（可选）**：在 `IntradayChart` 上包一层 `GestureDetector`/`Listener`，根据 localPosition 算索引再画竖线+横线+tooltip；或使用 fl_chart 的 `LineTouchData` 等。

按上述方案即可在**不增加任何新接口**的前提下，让分时图的时间轴和 Y 轴在观感上对齐别人行情软件。
