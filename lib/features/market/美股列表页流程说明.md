# 美股列表页整体流程

## 数据源优先级

| 场景 | 数据源 | 说明 |
|------|--------|------|
| 本地 DB 有数据 | SQLite tickers + quotes | 秒开，SQL 排序 |
| 本地 DB 无数据 | 缓存 → 内置 S&P 500 → 全量 Polygon | 后台同步全量列表到 DB |

## 流程概览

```
进入美股 Tab（全部）
  │
  ├─ 1. 读本地 DB
  │     ├─ 有数据 → 展示 tickers + quotes，SQL 排序
  │     │           ├─ 订阅 WebSocket 成交流（可视区域）
  │     │           ├─ 实时推送 → 更新 _quotes → 防抖写入 DB
  │     │           ├─ 定时 15 秒拉取可视区域报价
  │     │           └─ 定时 1 小时同步 tickers
  │     │
  │     └─ 无数据 ↓
  │
  ├─ 2. 缓存 / 内置 S&P 500 秒开
  │     └─ 展示 _allTickers
  │
  ├─ 3. 后台 syncTickers（DB 空时用全量美股列表）
  │     ├─ 有后端：stock_quote_cache → 缓存 → 内置
  │     ├─ DB 空：getAllUsTickers（Polygon）全量约 8000+ 条
  │     ├─ 写入 DB
  │     └─ 同步到服务器 tickers-upsert
  │
  ├─ 4. 合并 stock_quote_cache（补充 symbol/name，不替换）
  │
  ├─ 5. 拉取报价
  │     ├─ 可视区域：REST /api/quotes
  │     ├─ 订阅 WebSocket 成交流（可视区域）
  │     └─ 后台分批拉取其余（每批 500）
  │
  └─ 6. 持久化
        ├─ 报价更新 → persistQuotesToLocalDb
        └─ 本地缓存（TradingCache）
```

## 关键组件

| 组件 | 作用 |
|------|------|
| `MarketDb` | SQLite 存储 tickers + quotes |
| `MarketSyncService.syncTickers` | DB 空时用全量列表同步 |
| `RealtimeQuoteService` | WebSocket 成交流，订阅可视区域 |
| `_persistQuotesToCache` / `_debouncedPersistQuotes` | 报价写入 DB |
| `_loadQuotesForVisibleRange` | 按滚动位置拉取报价 |
| `_prefetchAllQuotesInChunks` | 后台分批拉取非可视区域 |

## 实时数据流

1. **REST**：首次加载、定时刷新、可视区域变化时拉取
2. **WebSocket**：订阅可视区域（约 40 个 symbol），推送成交价 → 更新 _quotes → 防抖 500ms 写入 DB
3. **排序**：有 DB 时用 SQL 排序；无 DB 时内存排序
